**关于 SOCD_Bridge** 
--------------------------------------------------------------------------------------------------
SOCD_Bridge 为键盘无默认驱动级SOCD，又想低成本使用SOCD功能的替代方案，使其适用于所有键盘！

SOCD_Bridge 是一款轻量级的开源工具，在系统托盘中运行，旨在追踪 WASD 按键的输入，同时不会干扰任何游戏文件。其主要功能是识别这些按键的按下状态，并自动释放之前已激活的命令，从而确保响应迅速且精确的输入处理。SOCD_Bridge 默认处理 WASD 按键，并允许您通过配置文件自定义按键绑定。

**SOCD_Bridge 功能特性**
--------------------------------------------------------------------------------------------------
- 简单易用
- 可随时激活或停用
- 轻量级且开源
- 通过系统托盘访问
- 兼容所有键盘
- 不与游戏文件交互
- 可通过右键菜单激活/停用
- 双击托盘图标即可禁用
- 内置及自定义键盘布局配置方案
- 按键粘滞功能：追踪已按下按键的状态
- 提升游戏中急停移动的精确度 
- 支持使用配置文件中的 ASCII 码重新绑定按键 
- 支持跨组共享的无限数量按键（默认 AD / WS）
- 使左右移动之间的切换更加流畅，避免输入冲突 
- 不使用 AutoHotkey 或类似工具，功能完全依赖 Windows API 函数

**SOCD_Bridge 工作原理**
--------------------------------------------------------------------------------------------------
- 当您按住 **"A"** 键时，SOCD_Bridge 会记住它。
- 如果您在按住 **"A"** 的同时按下 **"D"** 键，SOCD_Bridge 会自动为您释放 **"A"** 键。
- 同样，当您按住 **"D"** 时按下 **"A"**，SOCD_Bridge 也会释放 **"D"** 键。

**SOCD_Bridge 防止移动按键同时冲突（AD / WS）**

- 在许多 FPS 游戏中，同时按下 **"A"** 和 **"D"** 键通常会导致游戏识别到冲突的输入。SOCD_Bridge 会在检测到新的按键输入时自动释放之前按住的按键。
- 按键被分为两个不同的组：A/D 和 W/S。在每组中，**"A"** 会取消 **"D"**，反之亦然，**"W"** 和 **"S"** 也是如此。这些组互不干扰，各自独立工作。

**按键粘滞功能**

- 按键粘滞是一项追踪已按下按键状态的功能。例如，如果您按住 **"A"** 键并反复点击 **"D"** 键，每次按下 **"D"** 都会临时覆盖 **"A"** 键。当您释放 **"D"** 键后，只要您仍然按住 **"A"** 键，与 **"A"** 键关联的操作将会恢复。同样的原理也适用于先按住 **"D"** 再按 **"A"** 的情况。

> 🚨 <span style="color:red">【特别注意】SOCD_Bridge 及类似方案在某些游戏中已被禁止使用，使用前请注意，后续导致的任何资产损失与本项目无关</span>

**提升急停精确度**

- 当按下新的按键 (A/D) 和 (W/S) 时，自动释放之前按住的按键。
